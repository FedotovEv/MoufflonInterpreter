
Демонстрационная система поддержки многофайлового проекта в интерпретаторе Муфлона.
-----------------------------------------------------------------------------------

Выложенная здесь программа показывает, каким способом можно обеспечить в данном интерпретаторе
исполнение многомодульных комплексов, каждый модуль которых представляет собой отдельный
файл. Для этого программисту следует реализовать специализированный диспетчер модулей,
предоставляющий для лексического разборщика интерпретатора интерфейс, унаследованный от
parse::LexerInputEx. Что, собственно, в представленной здесь программе и сделано.

Реализованный в ней класс LexerFileInputExImpl предполагает, что имена подключаемых файлов
совпадают с именами соответствующих модулей, фигурирующих в директиве include, а хранятся они
в том же каталоге, где размещается начальный, стартовый файл проекта, имя которого передаётся
программе посредством командной строки. Формат вызова программы, следовательно, такой:

        MythonInclude.exe program_name

где program_name - стартовый файл программы.

Затем диспетчер по директиве include Муфлона переключает файлы-модули, предоставляя к считыванию
их содержимое в нужном порядке. 

К программе приложен примерный многофайловый Муфлон-проект, стартовый файл которого имеет имя
MainProgram.muf. Этот проект реализует сложную иерархическую систему многоуровневого включения
модулей друг в друга, позволяя при этом отслеживать цепочку их переключений. Наблюдая за картиной
вывода соответствующих сообщений на консоль, вы получите полное представление о процессе работы
таких программ.
